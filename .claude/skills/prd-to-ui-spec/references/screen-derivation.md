# 屏清单推导 · 组件映射 · 状态清单 · 数据绑定约定

## 1 如何从 PRD 推导屏清单

按优先级综合三条线索:

1. **应用层 / 界面线索**:PRD 总体设计里"应用层"列出的工作台/后台(如 配置工作台、评价工作台、回测分析台、标准迭代台、管理审计后台)往往直接对应一屏或一组屏。
2. **功能模块(§5)**:每个 `5.x` 模块的核心交互需要一个落地界面。把模块的 `[FR-*]` 归到对应屏。
3. **用户角色(§3)**:每个角色至少有一条主任务路径,确认每类角色都有可用界面。

产出**屏清单表**(先给用户确认):

| 屏 ID | 名称 | 主要角色 | 实现的 [FR-*] | 优先级 |
|-------|------|----------|---------------|--------|

约定:屏 ID `SC-1, SC-2 …` 全局唯一。一个大模块可拆多屏(如"评价与榜单"拆 榜单 + 复核台)。

## 2 组件映射速查（→ Shadcn/UI + 你的栈）

| UI 模式 | Shadcn / 库 | 说明 |
|---------|-------------|------|
| 数据表/榜单 | `Table` + **TanStack Table** | 排序/筛选/分页/虚拟滚动 |
| 表单 | `Form` + react-hook-form + **Zod4** | 校验与后端 DTO 对齐 |
| 弹层(编辑/确认) | `Dialog` / `Sheet` | 复杂表单用 Sheet 侧滑 |
| 分组折叠 | `Accordion` / `Collapsible` | 指标分组、长内容 |
| 选项卡 | `Tabs` | 同屏多视图切换 |
| 树形(指标树/分类) | `Accordion` 嵌套 / 自定义 + `dnd` | 拖拽排序 |
| 徽标/状态 | `Badge`(variant) | 档位/标记/否决 |
| 提示/警示 | `Alert`(default/destructive) | 否决横幅、错误 |
| 悬浮说明 | `Tooltip` / `HoverCard` | 计算式、口径释义 |
| 区间/打分 | `Slider` / `Input(number)` | 定性分、阈值 |
| 步骤流程 | 自定义 Stepper / `Tabs` | 导入向导、回测流程 |
| 时间线/留痕 | `ScrollArea` + 自定义 | 审计、操作历史 |
| 反馈 | `Toaster`(sonner) | 保存成功/失败 |
| 骨架 | `Skeleton` | 加载态 |
| 路由 | **TanStack Router** | `/path/$param` |
| 数据获取 | **TanStack Query** | useQuery / useMutation |

## 3 页面状态必填清单（逐项想过,不可漏）

| 状态 | 何时 | 设计要求 |
|------|------|----------|
| 加载中 | 拉取数据 | `Skeleton`,避免布局跳动 |
| 错误 | 接口失败 | `Alert` + 重试(Query retry) |
| 空 | 无数据/首次 | 空态插画 + 引导操作 |
| 无权限 | 角色不足(Sa-Token) | 整页/区块禁用或提示;只读角色隐藏写操作 |
| 数据缺失/未确认 | 抽取未质检、佐证缺失 | 标记 + 禁用下游动作(如未确认不计入排序) |
| 业务否决/异常态 | 一票否决、互斥 | destructive 横幅 + 降级展示 + 申诉/复核入口 |
| 成功反馈 | 写操作完成 | `Toaster` + 局部刷新 |

> 这张清单是设计说明相对"直接出图"的核心价值——图不会替你想这些,但开发必须实现它们。

## 4 数据绑定约定

- 每个动态元素写明:`UI 元素 | 建议接口/字段 | 回指[FR-*]`。
- 字段命名尽量对齐 `indicators.yaml`(指标/规则/上限/排名系数等)与后端 DTO。
- 统一注明:**接口形态以 OpenAPI 契约为准;前端类型经 orval 由契约生成;表单校验用与后端 DTO 对齐的 Zod schema**。
- 原型阶段先用"同形 mock 数据"驱动,联调时换 orval 类型。

## 5 追溯纪律

- spec 只能引用 PRD 已存在的 `[FR-*]/[NFR-*]`;用 `scripts/check_trace.py` 校验。
- 设计说明**不新增需求**。若推导界面时发现 PRD 缺需求,回 `prd-writer` 补 PRD,再回来出设计说明,保持单向追溯。
