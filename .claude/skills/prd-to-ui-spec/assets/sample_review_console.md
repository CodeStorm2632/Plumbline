# SC-1 评价复核台 · UI 设计说明

> 这是 PRD 与原型生成之间的"UI 契约"。逐屏描述布局、组件映射、状态、数据绑定与交互规则，并回指 `[FR-*]`（对齐真源 PRD-talent-eval@v1.0）。下一棒（Claude Design / Claude Code）据此生成 Shadcn 原型。

## 1 概述

- **目的**：让评审专家在单个申报人的"自动计分结果"上做人工复核——查看可解释明细、逐条质检加减分、处理负面清单、发起直通车、标记亮点、录入定性分，且全程留痕。
- **主要角色**：评审专家（`评审专家`）。`回测分析员/标准配置员` 只读。
- **入口路由**：`/review/$applicantId`（TanStack Router）。
- **一句话**：把"机器算的分"交给"人来判"，每一步都可解释、可追溯、可留痕。

## 4 组件映射（→ Shadcn/UI）

| 区块 | Shadcn 组件 | 备注 |
|------|-------------|------|
| 申报人头部条 | `Card` + `Badge` + `Separator` | 档位/标志性/直通车/亮点/否决用不同 `Badge` variant |
| 否决横幅 | `Alert`(variant=destructive) | 仅命中负面清单时出现 |
| 指标分组折叠 | `Accordion` / `Collapsible` | 按 标志性/核心/一般 分组 |
| 计分明细行表 | `Table`（**TanStack Table** 驱动） | 列：规则 \| 证据 \| 计算式 \| 得分 \| 佐证 \| 质检 |
| 计算式说明 | `Tooltip` / `HoverCard` | 悬浮显示 `25×100%=25`、排名系数等 |
| 佐证预览 / 材料定位 | `Sheet` / `Dialog` | 打开佐证原件、跳转到"成果作品第 n 行" |
| 行内质检 | `Button`组(认可) + `Dialog`(修正/驳回) | 修正/驳回打开带 `Textarea` 的表单 |
| 直通车 / 亮点 / 定性分 | `Dialog` 或 `Sheet` + `Form` | react-hook-form + **Zod4** 校验，理由必填 |
| 定性分输入 | `Slider` / `Input(number)` + `Form` | 取值限定在配置区间（见 §10 OQ-3） |
| 审计/留痕 | `ScrollArea` + 自定义时间线 | 操作人/时间/理由 |
| 保存反馈 | `Toaster`(sonner) | 成功/失败提示 |
| 加载骨架 | `Skeleton` | 见 §5 |

## 5 页面状态

| 状态 | 触发 | 表现 |
|------|------|------|
| 加载中 | 拉取评价结果 | 头部 + 明细区 `Skeleton` |
| 错误 | 接口失败 | `Alert` + [重试]（TanStack Query `retry`） |
| 无权限 | 非评审专家（Sa-Token） | 整页 `Alert`：无复核权限；只读角色隐藏所有操作按钮 |
| 未质检确认 | 数据未经质检 `[FR-5.3.2]` | 明细可见但标"未确认"，**禁用**计入排序，引导先质检 |
| 佐证缺失 | 某行无佐证 | 该行标 ⚠，`认可` 按钮禁用直至补佐证 |
| 已否决 | 命中负面清单 `[FR-5.1.8]` | 顶部 destructive 横幅；明细置灰为次要信息；保留[申诉]入口 |
| 空 | 该人尚无评价 | 空态插画 + [运行评价] |

## 7 交互与校验规则

- **逐条质检**：每行可 `认可 / 驳回 / 修正`。`驳回/修正` 必填理由（Zod required）；任一操作写审计（操作人/时间/理由）。`[FR-5.1.3]` `[NFR-4]`
- **质检门槛**：未确认数据不计入正式排序；页面明确区分"已确认/未确认"。`[FR-5.3.2]`
- **直通车**：仅授权角色可发起（权限规则见 OQ-2）；需理由 + 依据附件；提交进入审核流并留痕。`[FR-5.1.6]`
- **亮点**：标记为"额外推荐"候选，**不改总分**；理由入审计。`[FR-5.1.7]`
- **定性分**：在配置区间内打分（区间见 OQ-3），理由必填；明细标注"人工"来源。`[FR-5.1.3]`
- **负面清单**：命中须录认定依据来源；该人标"否决"、移出排序、保留申诉入口。`[FR-5.1.8]`
- **标志性高亮**：触发第一档的行加"标志性"徽标并置顶分组。`[FR-5.1.5]`

## 8 边界与异常

- 同一行重复质检：以最后一次为准，但**历史留痕全保留**。`[NFR-4]`
- 直通车/否决并存：否决优先，直通车不可越过一票否决。
- 大量明细行：表格虚拟滚动（TanStack Table + 虚拟化），避免长榜卡顿。
- 密级：佐证原件按密级控制可见性与下载水印。`[NFR-3]`

## 9 验收要点（本屏）

- 任一加减分行都能展开看到 指标ID/规则ID/证据/计算式/佐证/材料定位 六要素齐全。`[FR-5.1.3]`
- `修正/驳回/直通车/亮点/否决` 每个动作都生成一条可导出的审计记录。`[NFR-4]`
- 未质检确认的数据在 UI 上不可计入排序。`[FR-5.3.2]`

## 10 开放问题（继承 PRD）

- **[OQ-2]** 直通车发起权限/审核层级未定 → 影响"发起直通车"按钮的可见性与提交后流向。本屏先按"可发起—进审核—留痕"实现。
