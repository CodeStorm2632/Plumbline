# SC-3 角色管理 · UI 设计说明

> 维护角色与其菜单/权限授权，驱动动态 RBAC。授权变更即时失效权限缓存。

## 1 概述

- **目的**：管理员维护角色——列表、新建/编辑、软删除、给角色分配菜单/按钮权限。
- **主要角色**：`管理员`（`sys:role:read` + `sys:role:write`）。
- **入口路由**：`/sys/roles`。
- **一句话**：权限即数据，角色授权可视化维护、保存即生效。

```
┌──────────────────────────────────────────────┐
│ 角色管理                     [+ 新建角色]      │
├──────────────────────────────────────────────┤
│ 角色码 | 名称 | 备注 | 权限数 | 操作           │
│ 管理员 | 管理员| —    | 1(*)  | [编辑][授权][删]│
```

## 5 页面状态

| 状态 | 触发 | 表现 |
|------|------|------|
| 加载中 | 拉取列表 | `Skeleton` |
| 错误 | 接口失败 | `Alert` + [重试] |
| 空 | 无角色 | 空态 + [新建角色] |
| 无权限 | 无 `sys:role:write` | 隐藏写操作；无 read 整页提示 |
| 缺失 | 角色无授权 | 权限数显示 0 |
| 否决/受限 | 删除被引用角色 | 409 toast「角色已被用户引用」 |
| 成功 | 写/授权完成 | toast + 失效列表 query |

## 6 数据绑定

| UI 元素 | 来源（接口/字段） | 回指 |
|---------|------------------|------|
| 列表行 | `GET /api/sys/roles` → `id,code,name,remark,menu_ids` | `[FR-6.2.1]` |
| 新建 | `POST /api/sys/roles` `{code,name,remark}` | `[FR-6.2.2]` |
| 编辑 | `PUT /api/sys/roles/{id}` `{name,remark}` | `[FR-6.2.2]` |
| 删除 | `DELETE /api/sys/roles/{id}`（软删） | `[FR-6.2.3]` |
| 授权 | `POST /api/sys/roles/{id}/menus` `{menu_ids}` | `[FR-6.2.4]` `[NFR-6.3]` |

## 7 交互与校验规则

- **授权即时生效**：分配菜单后后端失效该角色权限缓存，下一次鉴权按新权限。`[NFR-6.3]`
- **引用守卫**：删除被用户引用的角色返回 409，前端提示先解绑。`[FR-6.2.3]`
- **按钮级权限**：无 `sys:role:write` 隐藏写操作；越权 403。`[NFR-6.4]`

## 8 边界与异常

- 角色码重复：后端 409，表单内联报错。
- 授权空集合：清空该角色全部权限（合法）。

## 9 验收要点（本屏）

- 给角色加/减菜单后，该角色用户的权限即时变化（缓存失效）。`[FR-6.2.4]` `[NFR-6.3]`
- 删除被引用角色被拒（409）。`[FR-6.2.3]`
