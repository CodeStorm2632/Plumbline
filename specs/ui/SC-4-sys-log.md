# SC-4 日志管理 · UI 设计说明

> 只读查询操作日志与登录日志，支持过滤分页，敏感信息脱敏展示。

## 1 概述

- **目的**：审计员/管理员查询操作日志（谁、何时、对什么做了什么）与登录日志（成功/失败/锁定）。
- **主要角色**：`管理员`、`审计员`（均有 `sys:log:read`）。
- **入口路由**：`/sys/logs`。
- **一句话**：所有写操作与登录可查、可追溯、脱敏展示。

## 2 需求追溯

| 区块 | 回指 |
|------|------|
| 操作日志查询（实体/操作人/时间） | `[FR-6.4.1]` `[NFR-6.4]` |
| 登录日志查询（用户名/结果/时间） | `[FR-6.4.2]` |
| 脱敏展示 | `[FR-6.4.3]` `[NFR-6.6]` |

## 3 信息架构 / 布局（低保真）

```
┌───────────────────────────────────────────────┐
│ 日志管理   [操作日志 | 登录日志]                │
│ 过滤：[操作人___][实体___][起][止]  [查询]      │
├───────────────────────────────────────────────┤
│ 时间 | 操作人 | 动作 | 实体 | 详情(脱敏)        │
│ …（分页）                                       │
└───────────────────────────────────────────────┘
```

## 4 组件映射（→ Shadcn/UI）

| 区块 | Shadcn 组件 | 备注 |
|------|-------------|------|
| 页签 | `Tabs` | 操作日志 / 登录日志 |
| 过滤 | `Input` + `DatePicker` | 操作人/实体/时间范围 |
| 列表 | `Table`（TanStack Table） | 只读，分页 |
| 状态徽标 | `Badge` | 登录成功/失败 |
| 骨架 | `Skeleton` | 见 §5 |

## 5 页面状态

| 状态 | 触发 | 表现 |
|------|------|------|
| 加载中 | 拉取日志 | `Skeleton` |
| 错误 | 接口失败 | `Alert` + [重试] |
| 空 | 无匹配日志 | 空态提示 |
| 无权限 | 无 `sys:log:read` | 整页提示无权限 |
| 缺失 | 字段为空 | 显示「—」 |
| 否决/受限 | — | 只读屏无写操作 |
| 成功 | 查询返回 | 渲染分页表格 |

## 6 数据绑定

| UI 元素 | 来源（接口/字段） | 回指 |
|---------|------------------|------|
| 操作日志行 | `GET /api/sys/logs/audit?entity_id&actor&start&end` → `ts,actor,action,entity_id,before,after` | `[FR-6.4.1]` |
| 登录日志行 | `GET /api/sys/logs/login?username&success&start&end` → `ts,username,success,ip,detail` | `[FR-6.4.2]` |
| 详情脱敏 | 自由文本中的手机号样式被掩码 | `[FR-6.4.3]` `[NFR-6.6]` |

## 7 交互与校验规则

- **只读**：本屏无写操作。
- **脱敏**：detail 等自由文本中的手机号样式后端已掩码，前端原样展示。`[FR-6.4.3]`
- **权限**：无 `sys:log:read` 整页无权限提示；越权 403。`[NFR-6.4]`

## 8 边界与异常

- 大量日志：分页 + 时间范围过滤；默认倒序最近优先。
- 时间范围非法：前端约束 start ≤ end。

## 9 验收要点（本屏）

- 操作日志可按操作人/实体/时间过滤。`[FR-6.4.1]`
- 登录日志含成功/失败、可按用户名过滤。`[FR-6.4.2]`
- 自由文本中的手机号被脱敏，不出现明文。`[FR-6.4.3]`
